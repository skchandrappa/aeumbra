## Deploy Backend to Railway (Recommended)

### Prerequisites
1. GitHub account
2. Railway account (free at railway.app)

### Steps

1. **Push your code to GitHub** (if not already done):
   ```bash
   git add .
   git commit -m "Prepare for deployment"
   git push origin main
   ```

2. **Sign up at Railway**:
   - Go to https://railway.app
   - Sign up with GitHub
   - New Project â†’ Deploy from GitHub repo

3. **Configure Railway**:
   - Select your `aeumbra` repository
   - Set root directory to `pythonbackend`
   - Add environment variables:
     ```
     DATABASE_URL=<auto-generated by Railway>
     SECRET_KEY=your-secret-key-here
     ALGORITHM=HS256
     ACCESS_TOKEN_EXPIRE_MINUTES=30
     REFRESH_TOKEN_EXPIRE_DAYS=7
     ENVIRONMENT=production
     DEBUG=False
     ```

4. **Get your production URL**:
   - Railway provides: `https://your-app.railway.app`
   - Use this URL in your frontend configuration

---

## Deploy Frontend to Vercel

1. **Go to Vercel**: https://vercel.com
2. **Import GitHub repository**
3. **Set environment variable**:
   - `REACT_APP_API_URL=https://your-backend.railway.app`
4. **Deploy!**

---

## Update Frontend API URL

In `auembranodeserv/.env`:
```env
REACT_APP_API_URL=https://your-backend.railway.app
```

For production, create `.env.production`:
```env
REACT_APP_API_URL=https://your-backend.railway.app
```

## Troubleshooting

**Issue**: Database connection error
- **Solution**: Check `DATABASE_URL` format and SSL mode

**Issue**: Build fails
- **Solution**: Verify all dependencies in `requirements.txt`

**Issue**: CORS errors
- **Solution**: Add your frontend URL to `allow_origins` in `main.py`
